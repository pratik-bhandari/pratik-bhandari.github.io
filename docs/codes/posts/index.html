The first thing that I do for any data analyses project is to create a remote git repository. It can be in github, or in gitlab. (Do people still use bitbucket?) Then I create an R project, in R Studio, and connect it with the remote git repository. When you click on ‘New Project’ on the top right drop down menu, R Studio give you three choices and one of them is to Checkout a project from a version control repository. … …

Some of the most frequently used git commands…

git push -u origin master #to push to the remote repo for the first time
git checkout -b NewBranchName
git push -u origin BranchIamWorkingOn #to push from local repo to remote repo if the remote repo doesn't have this `BranchIamWorkinOn` setup up already
git branch -D BranchToBeDeleted

To merge

git checkout main #First switch to the main/master branch
git merge BranchIwantToMerge #The branch still exists after merging
